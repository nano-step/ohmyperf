**Why v4 differs from v3 — practical impact**:

- **No `tailwind.config.ts` required**; tokens live in CSS via `@theme`. The TS config above is *only* for content-scanning hints to faster editor tooling. v4 also auto-detects sources from your `@import` site.
- **PostCSS plugin changed**: it's `@tailwindcss/postcss` (a separate package), not `tailwindcss` directly. v3 muscle-memory will fail silently if you forget.
- **`@apply` still works**, but shadcn's v4-compatible templates use raw CSS custom properties (`var(--color-primary)`) more than utilities — expect generated components to look different from v3 shadcn output.
- **OKLCH colors** are the default. They render correctly in all modern browsers but produce different hex equivalents in screenshots compared to v3's RGB defaults.
- **No `darkMode: 'class'` config** — variant is built-in. Toggle by adding `class="dark"` to `<html>` (handled by `next-themes`).
- **Build speed**: v4 is significantly faster (~10×) — but the **beta** designation matters. Pin exact (`4.0.0-beta.8` or whatever's current at impl time); see REVIEW.md N1. If it breaks mid-development, fallback to v3.4.x is a 30-min revert.

```ts
import { z } from 'zod';

const PRIVATE_HOSTS = new Set(['localhost', '127.0.0.1', '0.0.0.0', '::1']);
const PRIVATE_RANGES = [
  /^10\./,
  /^192\.168\./,
  /^172\.(1[6-9]|2\d|3[0-1])\./,
  /^169\.254\./,
  /^127\./,
  /^fc/i, /^fd/i,    // IPv6 ULA
  /^fe80/i,          // IPv6 link-local
];

export type PrivateHostHint =
  | { kind: 'none' }
  | { kind: 'loopback'; host: string }
  | { kind: 'private'; host: string };

export function detectPrivateHost(rawUrl: string): PrivateHostHint {
  try {
    const u = new URL(rawUrl);
    const host = u.hostname.toLowerCase();
    if (PRIVATE_HOSTS.has(host)) return { kind: 'loopback', host };
    if (PRIVATE_RANGES.some((rx) => rx.test(host))) return { kind: 'private', host };
    return { kind: 'none' };
  } catch {
    return { kind: 'none' };
  }
}

export const urlFormSchema = z.object({
  url: z
    .string()
    .trim()
    .min(1, 'Enter a URL to measure')
    .max(2048, 'URL too long')
    .refine(
      (v) => {
        try {
          const u = new URL(v);
          return u.protocol === 'http:' || u.protocol === 'https:';
        } catch {
          return false;
        }
      },
      { message: 'Must be a valid http(s) URL' },
    ),
});

export type UrlFormValues = z.infer<typeof urlFormSchema>;
```

```ts
import { env } from './env';

export type Backend =
  | { kind: 'extension'; extensionId: string; version: string }
  | { kind: 'runner'; baseUrl: string; version: string; engine?: string; browser?: { source: string; version: string } }
  | { kind: 'none' };

const RUNNER_HOSTS = [
  `http://127.0.0.1:${env.NEXT_PUBLIC_RUNNER_PORT}`,
  `http://localhost:${env.NEXT_PUBLIC_RUNNER_PORT}`,
];

const DETECTION_TIMEOUT_MS = 800;        // D2

/** Run extension + runner pings in parallel. Returns first success, or 'none'. */
export async function detectBackend(signal?: AbortSignal): Promise<Backend> {
  if (typeof window === 'undefined') return { kind: 'none' };

  const ac = new AbortController();
  const linkedSignal = signal
    ? mergeSignals([ac.signal, signal])
    : ac.signal;
  const timer = setTimeout(() => ac.abort('detection timeout'), DETECTION_TIMEOUT_MS);

  try {
    const [extResult, runnerResult] = await Promise.allSettled([
      pingExtension(linkedSignal),
      pingRunner(linkedSignal),
    ]);

    if (extResult.status === 'fulfilled' && extResult.value) {
      return extResult.value;             // extension wins per D2 preference order
    }
    if (runnerResult.status === 'fulfilled' && runnerResult.value) {
      return runnerResult.value;
    }
    return { kind: 'none' };
  } finally {
    clearTimeout(timer);
  }
}

async function pingExtension(signal: AbortSignal): Promise<Backend | null> {
  const id = env.NEXT_PUBLIC_EXTENSION_ID;
  const runtime = window.chrome?.runtime;
  if (!id || !runtime || typeof runtime.sendMessage !== 'function') return null;

  return new Promise<Backend | null>((resolve) => {
    let settled = false;
    const onAbort = () => {
      if (!settled) {
        settled = true;
        resolve(null);
      }
    };
    signal.addEventListener('abort', onAbort, { once: true });

    try {
      runtime.sendMessage(id, { type: 'ohmyperf/ping' }, (response) => {
        if (settled) return;
        settled = true;
        signal.removeEventListener('abort', onAbort);

        // chrome.runtime.lastError is set when extension is not installed or unreachable.
        if (runtime.lastError) {
          resolve(null);
          return;
        }
        if (
          response &&
          typeof response === 'object' &&
          'ok' in response &&
          (response as { ok: unknown }).ok === true &&
          'version' in response &&
          typeof (response as { version: unknown }).version === 'string'
        ) {
          resolve({
            kind: 'extension',
            extensionId: id,
            version: (response as { version: string }).version,
          });
        } else {
          resolve(null);
        }
      });
    } catch {
      // Synchronous throw (e.g. Firefox doesn't expose chrome.runtime).
      if (!settled) {
        settled = true;
        resolve(null);
      }
    }
  });
}

async function pingRunner(signal: AbortSignal): Promise<Backend | null> {
  // Try both 127.0.0.1 and localhost — PNA may treat them differently in some Chrome versions.
  for (const baseUrl of RUNNER_HOSTS) {
    try {
      const res = await fetch(`${baseUrl}/api/health`, {
        method: 'GET',
        mode: 'cors',
        credentials: 'omit',
        cache: 'no-store',
        signal,
      });
      if (!res.ok) continue;
      const body = (await res.json()) as { ok?: boolean; version?: string; engine?: string; browser?: { source: string; version: string } };
      if (body.ok === true && typeof body.version === 'string') {
        return {
          kind: 'runner',
          baseUrl,
          version: body.version,
          engine: body.engine,
          browser: body.browser,
        };
      }
    } catch {
      // Network error, CORS rejection, PNA refusal — try next host or fall through to 'none'.
    }
  }
  return null;
}

function mergeSignals(signals: AbortSignal[]): AbortSignal {
  const ac = new AbortController();
  for (const s of signals) {
    if (s.aborted) {
      ac.abort(s.reason);
      return ac.signal;
    }
    s.addEventListener('abort', () => ac.abort(s.reason), { once: true });
  }
  return ac.signal;
}
```

**Edge cases handled**:
- Firefox/Safari (no `window.chrome.runtime`) → `pingExtension` returns `null` immediately.
- Extension installed but service worker dormant → first `sendMessage` wakes it; 800ms timeout covers worker boot.
- Runner reachable on `127.0.0.1` but `localhost` resolves to IPv6 `::1` only → loops both hosts.
- `chrome.runtime.lastError` is *only* readable inside the callback; checking after promise resolution loses it (handled).
- `AbortSignal` linkage propagates parent cancellation (e.g. user navigates away mid-detection).

**Where chrome typings live**: `apps/website/types/global.d.ts`, not `lib/`, so they're ambient. Excluded from emit by being in `types/` directory and referenced via `tsconfig.json` `include` (no need to import).

## M. CSP details

**Exact meta tag** (canonical form, lives in `app/layout.tsx`):
```
default-src 'self';
connect-src 'self' http://localhost:5174 http://127.0.0.1:5174;
img-src 'self' data: https:;
script-src 'self';
style-src 'self' 'unsafe-inline';
font-src 'self' data:;
frame-ancestors 'none';
base-uri 'self';
form-action 'self';
object-src 'none'
```

**Why `'unsafe-inline'` for styles (and why this is acceptable)**:
- Tailwind utility classes via `class="…"` attributes are **not** inline styles and require no CSP grant.
- The `'unsafe-inline'` grant is needed for:
  1. **`next-themes` flash-prevention script** writes an inline `<style>` block to set initial theme before hydration. Without it, you get a dark-mode flash (FOUC).
  2. **Next.js itself** injects a small `<style>` block for the build-time CSS modules ordering when `output: 'export'` is on.
  3. **Inline `style={{...}}` props** on any React component (we may use them for dynamic uPlot canvas sizing in γ).
- The **risk model**: CSP's primary protection is `script-src` (against XSS). `style-src 'unsafe-inline'` only allows CSS injection, which is dramatically lower risk. CSS expression injection (which was the original concern) is dead in modern browsers.
- **Alternative (don't bother)**: nonce-based CSP requires server-rendered nonces per request. Doesn't work with `output: 'export'` (build-time emits a single static HTML). Would require either (a) CDN edge nonce injection or (b) hash-based CSP where every inline `<style>` block is whitelisted by SHA-256. Both are 2-3 day investments and gain marginal security for a static SPA where the origin is fully trusted to ship its own styles.

**`connect-src` justification**:
- `'self'` → required for client-side fetches of `_next/static/*` chunks (Next 15 lazy-loads them).
- `http://localhost:5174` + `http://127.0.0.1:5174` → runner API (`/api/health`, `/api/measure`, SSE on `/api/jobs/:id/events`). Both forms necessary because PNA may treat them as distinct origins on some Chrome versions.
- **NO `https://ohmyperf.dev`** — that's the SPA origin itself; covered by `'self'`. Listing it explicitly is redundant and signals confusion.
- **NO extension protocols** (`chrome-extension://...`) — extension messaging uses `chrome.runtime.sendMessage`/`connect`, which are runtime APIs not subject to CSP `connect-src`. Verified in Chromium source.

**Watch-outs**:
- **`upgrade-insecure-requests`** intentionally omitted. We want `http://localhost:5174` to NOT be upgraded to HTTPS (runner is local, has no cert). Including this directive would break runner connectivity.
- **`worker-src`** not specified → falls back to `script-src 'self'` per spec. If Web Workers are added in γ (e.g. for CSV export off-thread), this needs explicit `'self'`.
- **CSP via `<meta>` has limitations**: `frame-ancestors`, `report-uri`, and `sandbox` are header-only per spec but most browsers tolerate them in meta. We list `frame-ancestors 'none'` for defense-in-depth; the real protection happens at the CDN/Pages config level — document in `docs/measurement-spa-deploy.md`.

## N. Bundle budget enforcement

**Wiring** — `@next/bundle-analyzer` is already in `package.json` (Section B); enable via `ANALYZE=true next build`. Output: `.next/analyze/client.json`, `.next/analyze/server.json`.

**N.1 — CI script** `apps/website/scripts/check-bundle-budget.mjs`:
```js
#!/usr/bin/env node
/* eslint-disable no-console */
import { readFile } from 'node:fs/promises';
import { join } from 'node:path';

const BUDGETS = {
  '/':               { firstLoadJsKb: 150 },      // D11
  '/measure':        { firstLoadJsKb: 200 },      // γ phase budget
  '/report/[[...id]]': { firstLoadJsKb: 250 },    // γ phase budget
};

const root = process.cwd();
const reportPath = join(root, '.next', 'app-build-manifest.json');
const buildOutputPath = join(root, '.next', 'build-manifest.json');

// Next 15 emits a per-route summary in `.next/.../build-stats.json` when
// experimental.outputFileTracing is on; analyzer JSON is more reliable.
const analyzerJsonPath = join(root, '.next', 'analyze', 'client.json');

async function loadJson(p) {
  try { return JSON.parse(await readFile(p, 'utf8')); }
  catch (e) { console.error(`Could not read ${p}: ${e.message}`); process.exit(2); }
}

const stats = await loadJson(analyzerJsonPath);
// Analyzer JSON shape: top-level array of { label: chunkName, gzipSize, parsedSize, statSize, groups: [...] }
// We sum gzipSize for chunks loaded by each app route.

const manifest = await loadJson(buildOutputPath);  // route → [chunks]

let failed = false;
for (const [route, budget] of Object.entries(BUDGETS)) {
  const routeChunks = manifest.pages?.[route] ?? manifest.rootMainFiles ?? [];
  const gzipBytes = stats
    .filter((chunk) => routeChunks.some((rc) => chunk.label.endsWith(rc.split('/').pop())))
    .reduce((acc, chunk) => acc + (chunk.gzipSize ?? 0), 0);
  const gzipKb = Math.round(gzipBytes / 1024);
  const status = gzipKb <= budget.firstLoadJsKb ? '✅' : '❌';
  console.log(`${status} ${route} → ${gzipKb}KB (budget ${budget.firstLoadJsKb}KB)`);
  if (gzipKb > budget.firstLoadJsKb) failed = true;
}

if (failed) {
  console.error('\nBundle budget exceeded. See .next/analyze/client.html for details.');
  process.exit(1);
}
```

## P. Open questions / implementer gotchas

**Compatibility matrix (as of 2026-05, pinned exactly)**:

| Stack item | Pinned version | Notes |
|---|---|---|
| Node | 20.x LTS (engines.node from root `package.json:16`) | Next 15 supports 18.18+ but 20 is project floor |
| Next.js | `15.1.4` | Stable. Patch latest in 15.1.x line is fine; minor bumps (15.2, 15.3) require re-test of catch-all. |
| React + React-DOM | `19.0.0` | GA. Some shadcn primitives still use legacy refs — works but produces dev warnings; ignore. |
| `@types/react` | `19.0.4` | Must match React major. **Workspace catalog at `pnpm-workspace.yaml:19-20` still pins 18.x — direct-pin in website package overrides, but plan to bump catalog in β.3 to avoid drift in other apps.** |
| Tailwind | `4.0.0-beta.8` | Beta. Pin exact (N1 in REVIEW). If stable 4.0.0 ships during β phase, upgrade in a single dedicated commit. |
| `@tailwindcss/postcss` | match Tailwind | Don't pin to `4.x`; pin same version as tailwindcss. |
| shadcn CLI | `canary` channel for v4 templates | Stabilizes when Tailwind v4 ships. |
| next-intl | `3.26.x` | v3 line. v4 has different middleware patterns; not relevant for static export. |
| TypeScript | `^6.0.3` (root) | Verify all libs ship .d.ts compatible with TS 6 strict mode. `next-intl` and `react-hook-form` are confirmed. |

**Static export breakage surface — explicit refuses**:

1. **Server Actions in form submission** — REFUSE. `<form action={serverAction}>` is incompatible with `output: 'export'`. Build will fail or actions will no-op at runtime. The URL form uses `onSubmit` + client navigation only.
2. **`app/api/*` route handlers** — REFUSE. Same reason. There are no API routes; all backend calls go to the runner (`http://localhost:5174`) or extension messaging.
3. **`<Image src="https://..."/>`** — REFUSE optimization. `images: { unoptimized: true }` is set; remote images load as `<img>` tags. For the OG image (`/og-image.png`), it's a static asset, so no issue.
4. **`fetch()` with `next: { revalidate }` options** — silently ignored. Avoid mentioning ISR/SSG-with-revalidation in code; it's confusing in this mode.
5. **`generateMetadata` returning dynamic data per slug** — works at build only with explicit `generateStaticParams`. For `/report/[[...id]]/`, we return `[]`, so dynamic metadata can't be derived from id. Either accept generic metadata for all report pages (acceptable v1 — no SEO concern, reports are private) or move metadata into `document.title` updates client-side.
6. **`cookies()`, `headers()` in Server Components** — REFUSE. Build-time only; values are static. We don't use them.
7. **next-intl middleware-based locale routing** — REFUSE. Handled by client provider per Section L.

**Optional future considerations** (out of scope for β, log as follow-ups):
- Migrate to `[locale]` segment if VI copy gains SEO priority (~4h).
- Add `worker-src 'self'` to CSP when γ introduces off-thread CSV export.
