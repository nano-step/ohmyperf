## D1. Cache core — schema + bundle hash + HMAC + blob store

- [ ] D1.1 `packages/replay-cache/src/cache/schema.ts`: define `Manifest`, `CacheEntry`, `BlobReference`, `BundleHashInputs`, error types (`ReplayCacheError` with `code` discriminator: `'REPLAY_PATH_TRAVERSAL' | 'REPLAY_HMAC_MISMATCH' | 'REPLAY_BUNDLE_INCOMPLETE' | 'REPLAY_REQUEST_BODY_MISMATCH' | 'REPLAY_CACHE_MISS'`).
- [ ] D1.2 `packages/replay-cache/src/cache/bundle-hash.ts`: implement `canonicalizeUrl(rawUrl, stripQueryRegex)`, `canonicalizeHeaders(headers)` (lowercase names, drop excluded list), `computeBundleHash(entries)` (sort by `(method, urlCanonical)`, hash tuple list). Excluded header list as a `const`: `['date', 'x-request-id', 'set-cookie', 'last-modified', 'etag', 'server', 'x-trace-id', 'cf-ray']` plus regex `/^x-.*-(id|trace|debug)$/i`.
- [ ] D1.3 Unit test in `packages/replay-cache/src/cache/bundle-hash.test.ts`: sorted-order independence, excluded-header independence, query-param-order independence, method-changing-changes-hash, body-changing-changes-hash, line-endings-preserved-in-hash.
- [ ] D1.4 `packages/replay-cache/src/cache/hmac.ts`: `signManifest(manifest, bundleHash): hmac`, `verifyManifest(manifest, bundleHash, hmac): boolean`. Use `crypto.createHmac('sha256', bundleHash).update(canonicalJSON(manifest_without_hmac))`. Canonical JSON = stable-sorted keys.
- [ ] D1.5 `packages/replay-cache/src/cache/blob-store.ts`: `writeBlob(cacheDir, body): Promise<{ sha256: string, encoding: 'gzip' }>` (gzip level 6, write to `blobs/<sha256>.gz` if not exists), `readBlob(cacheDir, sha256): Promise<Buffer>`. Filenames strictly `^[0-9a-f]{64}\.gz$`. Validate on read.
- [ ] D1.6 `packages/replay-cache/src/cache/manifest.ts`: `writeManifest(cacheDir, manifest)`, `readManifest(cacheDir): Promise<Manifest>` — readManifest verifies HMAC and rejects path-traversal in `bodyRef` fields. File mode `0600`.
- [ ] D1.7 Cross-machine determinism test: `packages/replay-cache/src/cache/bundle-hash-cross-machine.test.ts` — generate a fixture of synthetic entries with header maps in randomized order, hash them, compare against a checked-in golden hash. Run on CI matrix `[ubuntu-22.04, macos-14, ubuntu-22.04@node24]`.

## D2. Recorder — CDP Network listener + parallel response fetcher

- [ ] D2.1 `packages/replay-cache/src/recorder/network-listener.ts`: subscribe to `Network.requestWillBeSent`, `Network.responseReceived`, `Network.loadingFinished`, `Network.loadingFailed` on a `CDPSessionLike` (the existing abstraction in `packages/core/src/types.ts`). Build an in-memory `Map<requestId, PartialEntry>`. Drop entries for non-document protocols (data:, blob:, javascript:, chrome-extension:).
- [ ] D2.2 `packages/replay-cache/src/recorder/response-fetcher.ts`: on `loadingFinished`, queue a `Network.getResponseBody({ requestId })` call. Limit concurrency to 16 with a simple semaphore (no library). Per-entry try/catch: failures logged, entry omitted from the bundle, degradation pushed.
- [ ] D2.3 `packages/replay-cache/src/recorder/recorder-factory.ts`: export `createRecorder(opts: { cacheDir, redactStorageState? }): RecorderAttachment` where `attach(session, ctx) => Promise<RecorderHandle>`. Handle has `flush() => Promise<{ bundleHash, captureFingerprint, degradations }>`.
- [ ] D2.4 During flush: collect all entries, compute body sha256 per entry, dedupe blobs by sha256 (large bodies share storage), call `computeBundleHash`, write inline bodies into entry JSONs and large bodies into blob store, write manifest with HMAC. Frame-id stored per-entry for diagnostics only (not part of hash).
- [ ] D2.5 Recorder integration with engine.ts (`packages/core/src/engine.ts`): branch on `opts.replay?.mode === 'record'`; iterate `pageCtx.attachedFrames` to call `recorder.attach(frame.session, { isRoot: false, frameId })` for OOPIFs; flush after `pageCtx.close()`. Emit `report.meta.bundleHash` and `report.meta.replayMode = 'record'`.
- [ ] D2.6 storageState capture step in engine: after the run, before flush, call `pageCtx.context().storageState()` (or driver-provided equivalent), pass through `opts.replay.redactStorageState ?? defaultRedactStorageState`, write to `<cacheDir>/<bundleHash>/storage-state.json` mode `0600`.
- [ ] D2.7 Record-mode degradation: engine adds `{ capability: 'replay-record-overhead', severity: 'warn', detail: '...' }` to `report.meta.degradations` on every record run. CLI emits stderr warning at run start. MCP includes top-level `warning` field.
- [ ] D2.8 Unit/integration test `packages/replay-cache/src/recorder/recorder.test.ts` against a fake `CDPSessionLike` driving recorded fixtures of `requestWillBeSent` + `responseReceived` + `loadingFinished` events; asserts manifest contents, body sha256 dedup, degradation emission on `getResponseBody` failure.

## D3. Player — Fetch.fulfillRequest handler + OOPIF auto-attach + miss policy

- [ ] D3.1 `packages/replay-cache/src/player/miss-policy.ts`: pure function `resolveMiss(url, policy): 'fail' | 'allow-live' | 'allow-empty'`. Engine wires interactive CLI prompt → policy via a callback in `opts.replay.onCacheMiss`.
- [ ] D3.2 `packages/replay-cache/src/player/fetch-handler.ts`: install `Fetch.enable({ patterns: [{ urlPattern: '*' }], handleAuthRequests: false })` on a session. On `Fetch.requestPaused`, look up the entry by canonical URL + method; on match, verify request body for non-GET/HEAD/OPTIONS (canonical JSON compare for `application/json`, byte compare otherwise); fulfill via `Fetch.fulfillRequest` with stored response code, headers, base64-encoded body (from inline or blob).
- [ ] D3.3 On match for non-GET methods with mismatched body: `Fetch.failRequest({ errorReason: 'AccessDenied' })` and push `{ capability: 'replay-request-body-mismatch', url, ... }` to a player-scoped degradations buffer (drained into `report.meta.degradations` at run end).
- [ ] D3.4 On miss: consult policy. `'fail'` → `Fetch.failRequest({ errorReason: 'NameNotResolved' })` + degradation `{ severity: 'error' }`. `'allow-live'` → `Fetch.continueRequest` + degradation `{ severity: 'warn' }`. `'allow-empty'` → `Fetch.fulfillRequest({ responseCode: 200, body: '' })` + degradation.
- [ ] D3.5 `packages/replay-cache/src/player/player-factory.ts`: `createPlayer(opts: { cacheDir, onCacheMiss }): PlayerAttachment` — `attach(session, ctx)`. Loads manifest ONCE on first attach (caches in closure); verifies HMAC ONCE; subsequent `attach` calls reuse the parsed manifest.
- [ ] D3.6 OOPIF auto-attach: in engine.ts before `pageCtx.goto`, call `rootSession.send('Target.setAutoAttach', { autoAttach: true, waitForDebuggerOnStart: true, flatten: true })`. Subscribe to `Target.attachedToTarget` event; for each new session, call `player.attach(newSession, { isRoot: false, frameId })` and THEN `newSession.send('Runtime.runIfWaitingForDebugger')`. Order matters: Fetch.enable MUST complete before `runIfWaitingForDebugger` resolves.
- [ ] D3.7 SW bypass: in engine.ts before any player attachment, call `Network.setBypassServiceWorker({ bypass: true })` on every session. Set `report.meta.swBypassed = true` if any session was so configured.
- [ ] D3.8 Plugin compatibility check: at player attach time, query the plugin runtime for any plugin declaring `requires: 'fetch-domain'` (a new capability flag). If any present, refuse to enter replay with structured error `{ code: 'REPLAY_PLUGIN_FETCH_CONFLICT', plugins: [...] }` listing offenders.
- [ ] D3.9 Integration test `packages/replay-cache/src/player/player.test.ts`: against a fake session, drive `Fetch.requestPaused` events, assert `Fetch.fulfillRequest` is called with correct params for matches, `Fetch.failRequest` for misses with `'fail'` policy, request-body-mismatch detection.
- [ ] D3.10 Engine integration test `packages/core/src/engine.replay.test.ts`: round-trip a fixture through record → flush → replay against the same `MeasureOptions` (calibration locked) and assert resulting Report's network-driven metrics (LCP, FCP) are within ±5 ms of the original record run.

## D4. Reproducer codegen + CLI flags + MCP args + report integration

- [ ] D4.1 `packages/replay-cache/src/codegen/fingerprint.ts`: `computeCaptureFingerprint({ bundleHash, calibration, viewport, userAgent, chromiumMajorVersion, nodeMajorVersion }): string` — sha256 hex.
- [ ] D4.2 `packages/replay-cache/src/codegen/template.ts`: a multi-line string template emitting a TypeScript script. Embeds: `BUNDLE_HASH`, `CAPTURE_FINGERPRINT`, `CACHE_DIR_RELATIVE`, calibration constants object, viewport, UA, headless flag. Imports `chromium` from `playwright`, `attachPlayer` + `loadManifest` from `@ohmyperf/replay-cache/standalone`. On run: load manifest, recompute fingerprint, compare; on match launch Chromium with calibration, attach player, navigate, dump a fresh `report.json` adjacent to the script.
- [ ] D4.3 `packages/replay-cache/src/codegen/emit.ts`: `writeReproducer(reportDir, args): Promise<{ scriptPath, configPath }>` — writes `reproduce.ts` and `reproduce.config.json` (containing `{ bundleHash, captureFingerprint, calibration, cacheDirRelative }`). Mode `0644` for both (readable, executable via `pnpm tsx`).
- [ ] D4.4 Engine integration: after recorder flush, call `writeReproducer(reportDir, { bundleHash, captureFingerprint, calibration, cacheDir, viewport, userAgent, headless })`. Set `report.artifacts.reproduceScriptRef = './reproduce.ts'`.
- [ ] D4.5 `packages/replay-cache/src/standalone.ts`: minimal entrypoint exporting `attachPlayer(page, cacheDir, opts)`, `loadManifest(cacheDir)`, `verifyFingerprint(expected, computed)`. No imports from `recorder/` or `codegen/`. Tree-shake enforced by D0.2 ESLint rule.
- [ ] D4.6 CLI: add `--record`, `--replay`, `--replay-from <path>` flags in `apps/cli/src/commands/run.ts`. Map to `opts.replay` sub-object. Reject `--record --replay` with exit 1. `--replay-from` precedence: file extension `.json` → parse as report.json (read `meta.bundleHash`); `.ts` → parse sibling `.config.json`; directory → check for `manifest.json`. Ambiguous → exit 1.
- [ ] D4.7 CLI cache-miss prompt: when TTY and `--replay` without `--no-prompt`, register `opts.replay.onCacheMiss` callback that prompts interactively. On `y`, re-launch the entire run with `--record`. With `--no-prompt` or non-TTY, `onCacheMiss = 'fail'`.
- [ ] D4.8 CLI `cache prune` subcommand in `apps/cli/src/commands/cache-prune.ts`: parse size argument (`500MB`, `2GB`), enumerate `.ohmyperf/cache/<bundle_hash>/` directories with `stat` for size + mtime, sort by mtime ascending, delete whole bundles until total ≤ target. Print summary.
- [ ] D4.9 MCP: add `replay` arg to `apps/mcp-server/src/tools/measure.ts` input schema. Same shape as CLI's resulting sub-object. Always pass `onCacheMiss: 'fail'`. Propagate `bundleHash` and `reproduceScriptRef` into the structured response. Cache miss → `{ isError: true, error: { code: 'REPLAY_CACHE_MISS', missingUrl, hint } }`.
- [ ] D4.10 Reporter integration: every reporter under `packages/reporter-*/` (JSON, Markdown, HTML, JUnit, HAR, CSV, deck) reads `report.meta.bundleHash` and `report.artifacts.reproduceScriptRef` when present and surfaces them (Markdown footer, HTML "How to reproduce" section, JSON pass-through). Defensive `?.` chains for legacy reports.
